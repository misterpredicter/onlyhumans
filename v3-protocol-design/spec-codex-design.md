# Codex Builder: Design & Product Polish

## Mission

Make themo.live feel like a billion-dollar product. Every pixel, every interaction, every page transition should have taste. You're the design lead — improve everything you see.

## Context

- Project at: `Desktop/Hackathons/World Coinbase Hackathon x402/human-signal/`
- Web app in `web/` subdirectory (Next.js 15, App Router)
- Design system: DM Sans/Serif/Mono fonts, dark #0C0C0C nav, warm #F9F8F5 background, white cards with #E8E5DE borders
- Live at themo.live, deployed via Vercel from github.com/misterpredicter/worldxcoinbasexxmptproject.git
- PUSH YOUR CHANGES to the git remote when done

## What to Improve

Read the entire codebase first. Then improve with extreme taste:

### Visual Polish
- Typography hierarchy — make headlines hit harder, body text breathe
- Spacing and rhythm — consistent padding, margins, vertical rhythm
- Card design — shadows, borders, hover states that feel premium
- Color usage — the warm palette is right, but make accents pop where they should
- Animations — subtle page transitions, button feedback, loading states
- Mobile responsiveness — every page must work on a phone (judges may demo on mobile)

### Information Architecture
- Homepage should tell the story in 10 seconds: what is this, why does it matter, how do I use it
- /work page should feel like a feed you want to scroll, not a task list
- /docs should feel like Stripe's docs — clean, copy-paste-able, inviting
- Task detail pages should show results beautifully — charts, distributions, confidence

### Micro-interactions
- Vote submission should feel satisfying (animation, confirmation)
- Task creation should feel like launching something important
- World ID verification should feel like a gate opening, not a barrier

### The "90/9/1" Display
- Somewhere visible on every page (footer or subtle header badge): "90% to contributors. 9% to platform. 1% to founder."
- This should feel like a badge of honor, not fine print

## Constraints
- Don't break existing functionality
- Don't add dependencies without good reason
- Tailwind CSS is the styling system — use it
- Commit and PUSH to remote when done
